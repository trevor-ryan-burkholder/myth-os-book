# Myth-OS — Jekyll site

GitHub Pages site for the Myth-OS wiki, manuscript chapters, appendices, and companion articles.

## Layout

```
site/
├── _config.yml           Site config, navigation sections
├── Gemfile               Ruby deps (github-pages bundle)
├── _layouts/             default.html, page.html
├── _includes/            head.html, footer.html
├── assets/css/style.scss Stylesheet (deep oxblood palette)
├── index.md              Home page
├── wiki/                 Concept and topic pages
├── chapters/             Manuscript chapters, grouped by part
├── articles/             10-part companion series
├── appendices/           Appendices A–D
├── front-matter/         Note, who-for
└── back-matter/          Glossary
```

All Markdown pages are generated from the source files under `Myth-OS_Book/wiki/`, `Myth-OS_Book/manuscript/`, and `Myth-OS_Book/articles/`. Obsidian-style `[[wikilinks]]` are rewritten to standard Markdown links and Jekyll front matter is prepended.

## Local development

Install Ruby (any version ≥ 2.7), then:

```bash
cd site
bundle install
bundle exec jekyll serve
```

The site will be available at `http://127.0.0.1:4000/`.

## Deploying to GitHub Pages

1. Commit the `site/` folder to your repo.
2. In your repository on GitHub, go to **Settings → Pages**.
3. Under **Build and deployment**, choose **Source: Deploy from a branch**.
4. Select your branch and set the folder to `/site` (or move the contents of `site/` to `/docs` if you prefer GitHub Pages' default folder).
5. Save. The first build runs automatically.

If you want to deploy from a path other than `/docs` or root, set up the [official GitHub Pages Action](https://github.com/actions/jekyll-build-pages) and point it at `site/`.

## Regenerating content

The content under `wiki/`, `chapters/`, `articles/`, `appendices/`, `front-matter/`, and `back-matter/` was generated by `build_site.py`. To re-run it after edits to source:

```bash
python3 build_site.py
```

The script is idempotent: it wipes the generated section folders and rewrites them. Hand-written files outside those folders (`index.md`, `_layouts`, `_includes`, `assets`, `_config.yml`) are not touched.

## Customizing

- **Primary color.** Edit the CSS custom properties at the top of `assets/css/style.scss`. The current palette is built around `--c-primary: #6B0F1A` (deep oxblood).
- **Navigation.** Edit the `sections:` list in `_config.yml`.
- **Site title and metadata.** Edit the top of `_config.yml`.

## Notes

The `_config.yml` `url` and `baseurl` fields are blank by default. Set them once you know the production URL — e.g.:

```yaml
url: "https://yourname.github.io"
baseurl: "/myth-os"
```
