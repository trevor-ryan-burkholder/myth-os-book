The operating principle that governs all interaction with [The space](/wiki/the-space/): provide the least input that maintains stable [Contact](/wiki/contact/).

More input means more pressure. More pressure reduces [Access](/wiki/access/). Minimal input is not passivity. It is precision. The deliberate reduction of load to the level the system can hold without closing.

The system stays open under minimal input and closes under load, regardless of intent. This is why questions, repetition, and elaboration in contact with a [Presence](/wiki/presence/) register as pressure even when they feel like engagement. The system is not evaluating content. It is reading load.

The practical version is restraint applied to every step. Detection without forcing. Approach without leaning. Acknowledgment without follow-up. Influence without control. Each step uses the smallest input that works.
