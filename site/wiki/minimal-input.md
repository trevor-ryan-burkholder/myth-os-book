The operating principle that governs all interaction with [The space](/wiki/the-space/): provide the least input that maintains stable [Contact](/wiki/contact/).

More input introduces more pressure. More pressure reduces [Access](/wiki/access/). Minimal input is not passivity. It is precision. It is the deliberate reduction of load to the level the system can hold without closing.

The system stays open under minimal input and closes under load, regardless of the intention behind the load. This is why questions, repetition, and elaboration in contact with a [Presence](/wiki/presence/) register as pressure even when they feel like engagement. The system is not evaluating the content. It is reading the load.

The practical version of minimal input is restraint applied to every step in the sequence. Detection without forcing. Approach without leaning. Acknowledgment without follow-up. Influence without control. Each step uses the smallest input that works.
